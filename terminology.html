Operating Systems Terminology

TERMDEFINITION
operating systemsoftware that controls the overall operation of a computer including file storage and retrieval, program execution, and environment for program execution
job queuea system of organization of jobs waiting for execution that are resolved in a FIFO fashion
FIFOacronym for "first in, first out" -- meaning the first to be added is the first to be removed
interactive processinga feature of newer operating systems that allows the user to interact with the pgoram through remote terminals
time-sharinga feature that allows multiple users to access service simultaneously
multiprogrammingan example of how time-sharing is implemented where time is divided into intervals into which jobs are executed one at a time
application softwareprograms that perform tasks related to the machine's utilization
system softwarecompletes tasks that are common to computer systems in general which build the infrastructure needed for application software
utility softwareone category of system software that extend the capabilities of the operating system by doing tasks that are not included in the operating system but are important for computer installations
graphical user interface (GUI)the visual representation of files, programs and other objects can be manipulated in a way that is intuitive to users, using pictoral elements such as icons
shellan older user interface that allowed users to communicate with the machine using text via a keyboard and messages displayed on the monitor
kernelthe internal part of an operating system that contains software components that executre tha basis functions for computer installation
device driversoftware components that communicate with controllers and other peripheral devices to help execute tasks
file managera unit of the kernel that coordinates the machine's storage by recording the location and contents of all the files in storage and facilitates user access to and manipulation of those files
memory managera unit of the kernel that coordinates the machine's use of memory by allocating memory space as it is needed
virtual memorythe large "fictional" memory space created by the memory manager shifting files around between around main memory and mass storage so that the memory is always present in the main memory
boot strappingalso known as booting -- transfers the operating system from mass storage to main memory when the computer is turned on
ROMread-only memory -- a small portion of main memory which can be read but not changed and contains the computer's intitial program
deadlocka condition in which two or more processes can't be carried out because each one is waiting on the resources being used by the other
spoolingholding data for output at a later but more convenient time, especially when elements required for execution are busy
super user (or administrator)a person who has privileged access to an operating system and the ability to change settings of the operating system and grant or retract privileges for other users of the operating system
multi-factor authenticationthe requirement of mltiple pieces of evidence of user authentication to access an account
