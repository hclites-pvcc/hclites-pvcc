Programming Languages

TERM DEFINITION
machine language sequences of instructions that are encoded as numeric digits
debugging the task of locating and correcting errors within a machine language program
assembler a program that converts mnemonic expressions into machine language instructions
assembly language a mnemonic system for repesenting programs
translator a program that translates programs expressed as high level primitives into a machine-language program
compiler a translator that has to compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive
interpreter a program that executes instructions as they are translated instead of recording the translated version for future use
cross-platform software software that is independent of an operating system's design as well as the machine's hardware design, and therefore executable throughout a network
natural language languages that are spoken by people (such as English, Spanish, etc.)
formal language languages that are precisely degined by grammars
machine independence the property that allows statements in a programming language to be interpreted and compiled for multiple machines
high-level programming language a programming language that provides more abstractions for the programmer which makes it easier to write and read the program
object oriented programming (OOP) based on a paradigm in which a software system is viewed as a collection of units or objects, each of which can perform actions related to itself and other objects
scripting language langauges used to perform administrative tasks instead of developing complex programs
data types:
  • integer: numeric data that is whole numbers only
  • float: numeric data that might not be a whole number
  • string: any combination of characters that is shown as symbols
primitive data type includes integer, float, character, and Boolean data types
data structure the conceptual shape or arrangement of data
Python collection data structures
  • list: multiple items stored in a single variable, created using square brackets, which are ordered, changeable and allow duplicates
  • tuple: multiple items stored in a single variable, created with parentheses, which are ordered and unchangeable
  • set: multiple items stored in a single variable, created with curly brackets, which are unordered, unchangeable, and unindexed
  • dictionary: data stored in key:value pairs, created with curly brackets, which are changeable and do not allow duplicates
function a set of instructiosn for performing a task that can be used abstractly by other program units
local variable a variable declared within function scope
global variable a variable defined in the global scope which is not restricted to a particular part of a program
source program a program in its original form, before being translated to another language
object program the translated version of a program
